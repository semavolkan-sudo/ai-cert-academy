# architecture.md — AlpRoute

Sistem mimarisi, veri akışı ve moderasyon pipeline'ı. (Operasyon kuralları: `AGENTS.md`.)

## 1. Stack

| Katman | Teknoloji |
|---|---|
| Frontend | Saf HTML/JS (framework yok). `index.html` tek büyük inline `<script>` (~340KB). Leaflet.js harita. |
| Admin | `admin.html` ayrı panel (network-first, cache'lenmez). |
| PWA | `sw.js` service worker — app shell + harita tile cache. Cache key `alproute-shell-vN` (şu an **v87**); tile cache `alproute-tiles-vN` (şu an **v2**). |
| Backend | Supabase (Postgres + Auth + Storage + RLS). Proje ref: `ttcafvdswezftdnstlgs`. |
| Serverless | Vercel `/api/*` fonksiyonları (Node). GitHub `main` → otomatik deploy. |
| Harita | MapTiler (raster tile, key `CPSr0W3A34ufxGs6cGCs`) + Leaflet. Stil seçici: streets/hybrid/outdoor/satellite. |
| Çeviri | MyMemory (via `/api/translate` proxy, kaynak dil 'en'). |
| E-posta | Resend. Ödeme | Lemon Squeezy. |
| Moderasyon | OpenAI omni-moderation (metin + görsel) via `/api/moderate` ve `/api/moderate-photo`. |

## 2. Supabase tabloları (public şeması)

- **places** — mekânlar (OSM/Google içe aktarımı + kullanıcı eklemeleri). `status` (approved/pending/hidden), `report_count`.
- **place_photos** — topluluk fotoğrafları. Kolonlar: `id, place_key, place_name, url, status, rejection_reason, reviewed_at, reviewed_by, auto_approved, admin_seen, mod_debug(jsonb), created_at, user_id`.
- **place_ratings** — çok kategorili yıldız + `comment` + `status`.
- **place_prices** — fiyat + `note` + `status`.
- **photo_reports** — topluluk foto şikayeti: `id, photo_id→place_photos, user_id→auth.users, reason, created_at`, **unique(photo_id,user_id)**.
- **reports** — mekan şikayetleri (foto değil).
- **favorites**, **visits** — kullanıcı favori/ziyaret (RLS: `auth.uid()=user_id`).
- **profiles** — kullanıcı profili, `is_admin` (admin yetkisi buradan).
- **notifications** — kullanıcı bildirimleri (SECURITY DEFINER RPC `notify_user`).
- **nearby_posts**, **place_enrichment**, **translations**, **ui_glossary**, **app_sessions** — yardımcı tablolar.

### Trigger'lar (place_photos & ilgili)
- `trg_enforce_photo_status` (BEFORE INSERT, place_photos) — admin olmayan insert'lerde status'ü pending'e zorlar (client self-approve edemez). **Güvenlik temeli; bozulmamalı.**
- `trg_photo_report_hide` (AFTER INSERT, photo_reports) — bir foto için **3 farklı kullanıcı** şikayetine ulaşınca `place_photos.status='pending'`, `admin_seen=false`, `mod_debug += {community_hidden:true, report_count:N}`. SECURITY DEFINER.
- `notify-photo`, `notify_place`, `on_report_created` (mevcut) — bildirim/şikayet akışları.

## 3. Vercel `/api` endpoint'leri
- `/api/translate` — MyMemory proxy (texts, target).
- `/api/moderate` — METİN moderasyonu: çok dilli küfür blocklist (stem tabanlı, TR sondan eklemeli) + OpenAI omni. `{flagged, reason}`.
- `/api/moderate-photo` — GÖRSEL moderasyonu (sunucu-otoriteli). Supabase Database Webhook tetikler.
- `/api/route`, `/api/compare`, `/api/enrich`, `/api/interpreter` — rota/karşılaştırma/zenginleştirme yardımcıları.

### Vercel env değişkenleri
`OPENAI_API_KEY`, `SUPABASE_SERVICE_ROLE_KEY`, `MODERATE_WEBHOOK_SECRET`,
`MODERATE_IMG_THRESHOLD` (şu an **0.2**), opsiyonel `SUPABASE_URL`.

### vercel.json (repo kökü)
- `functions.maxDuration` — compare/enrich 60s, translate/route 30s. (moderate/moderate-photo
  ayar gerektirmez; Pro/fluid'de yüksek limit alır — log'da moderate-photo "Maximum: 5m".)
- `headers` — **cache bayatlamasını önler:**
  - `/sw.js` → `Cache-Control: public, max-age=0, must-revalidate` (tarayıcı sw.js'i her zaman
    taze kontrol eder → sürüm bump sonrası kullanıcı yeni sürümü kendiliğinden alır).
  - `/(.*).html` → aynı header (index.html/admin.html her istekte revalidate).
  - Kaynak: Vercel resmi PWA deploy rehberi. **Sürüm bump kuralı yine geçerli** — header
    sadece yeni sw.js'in hızlı görülmesini sağlar, cache'i boşaltan şey sürüm numarasıdır.

## 4. Moderasyon pipeline'ı (iki katmanlı + topluluk)

### A) Metin (yorum / fiyat notu) — client tetikli
`openRatingModal`/`openPriceModal` submit → `moderateContent(text)` → `/api/moderate`
→ temiz=`approved` (anında yayında), şüpheli/hata=`pending` (fail-closed). Admin →
**Yorumlar** sekmesinde orijinal + TR çeviri ile görür, onaylar/kaldırır.

### B) Foto — sunucu-otoriteli (webhook)
1. Kullanıcı foto yükler → `place_photos` **pending** olarak insert (trigger zorlar). Yükleyen kendi pending fotosunu anında görür.
2. Supabase **Database Webhook `moderate_photo`** (place_photos INSERT) → POST
   `https://alproute.com/api/moderate-photo` (header `x-webhook-secret`).
3. Endpoint (service-role): satırı DB'den **yeniden çeker** (payload'a güvenmez), OpenAI
   omni ile görseli moderate eder. Karar:
   - `flagged:true` VEYA riskli kategori skoru ≥ `MODERATE_IMG_THRESHOLD` → **pending** kalır.
   - temiz → **approved** + `auto_approved=true, admin_seen=false`.
   - karar verilemezse (anahtar/hata) → **pending** (fail-closed).
   - Her durumda `mod_debug = {flagged, decided, topCat, topScore, threshold}` yazılır (kalibrasyon).
   - **Bağımlılık:** `place-photos` Storage bucket'ı **public** olmalı (OpenAI görseli çekebilsin).
   - **Sınır:** omni görsel skorları DÜŞÜK olabilir (silah/kan ~0.08–0.12, masum ~0). Eşik
     ile şiddet/cinsel yakalanır ama *alaka/kalite* yakalanmaz → C katmanı şart.

### C) Topluluk şikayeti (alaka/kalite ağı)
Uygulamada her **kullanıcı fotosunda** 🚩 buton (`reportPhoto`→`openPhotoReportModal`→
`submitPhotoReport`). Sebepler: irrelevant/explicit/violence/private/spam. Girişli kullanıcı,
foto başına 1 kez (unique). **3 farklı kullanıcı** → trigger fotoyu pending yapar. Admin →
**Foto → Bekleyen**'de kırmızı **"🚩 Şikayetle gizlendi · N şikayet"** rozeti + sebep dökümü
(`loadPhotoReportReasons` → photo_reports'tan sayar) ile görür.

## 7. Dil / i18n sistemi (index.html)
Çok dilli, **çalışma-zamanı çeviri** sistemi (statik string sözlüğü değil).
- **`LANGS`** — ~56 dil + `orig` (Original). Üst bardaki **bayrak butonu** (`#flagBtn`) →
  `showLangPicker()`. Seçilen dil `reviewLang` (localStorage `uiLang`); kullanıcı açıkça
  seçince `alproute_lang_set='1'`.
- **Kaynak dil `SRC_LANG='en'`** — seed içerik/açıklamalar İngilizce. Hedef `orig`/`en` ise çeviri atlanır.
- **Çeviri backend'i: MyMemory**, `/api/translate` proxy üzerinden (batched `translateMany`,
  global `trCache`). Kullanıcı yorumları, notları, açıklamaları çevrilir.
  > NOT: backend maliyet için Claude API'den MyMemory'ye taşındı (koddaki yorum da "MyMemory" olarak düzeltildi).
- **`.tr-text` + `data-orig`** desenli elemanlar yerinde çevrilir. **MutationObserver** yeni eklenen
  çevrilebilir içeriği yakalar; çeviri titremesini (flicker) önleyen guard var.
- `viewerLang()` / `deviceLang()` / `detectLang()` / `isEnglish()` yardımcıları;
  giriş yapılmadıysa cihaz diline (`navigator.language`) düşer.
- **Nearby board oto-çeviri:** `autoTranslateNearby()` — yakın gönderileri viewer diline çevirir.
- **URL `?lang=xx`** parametresi açılışta dili ayarlar (paylaşım/deep-link ile).

## 8. Oturum (session) yönetimi & hesap durumu (index.html)
- **Eşzamanlı oturum guard'ı:** hesap başına **max 3 aktif cihaz** (`SESSION_MAX=3`).
  - `sessionGuardStart()` → `supa.rpc('register_session', {p_session_id, p_ua, p_max})`.
  - `startHeartbeat()` 45sn interval (+ `visibilitychange`) → `supa.rpc('heartbeat_session', {p_session_id})`.
    Dönüş `false` ise `sessionKicked()` → heartbeat durur + "3 aktif cihaz limiti" ekranı.
  - Manuel çıkış: `sessionGuardEnd()` → `supa.rpc('end_session', {p_session_id})` slotu boşaltır.
  - `_sid` saklandığı için **aynı tarayıcıda refresh yeni cihaz sayılmaz**. Backing tablo: `app_sessions`.
  - TTL/expiry sunucu tarafı RPC'lerde. (Geçmiş bug: eski 3-dakikalık aktif pencere meşru kullanıcıları
    atıyordu → token ömrüne hizalı daha uzun TTL ile düzeltildi.)
- **Hesap durumu:** `profiles.status='inactive'` (admin hariç) → `accountInactive()` → `showDeactivated()`.
  Admin → Users sekmesinden Active/Inactive yapar.
- **Pro durumu:** `profiles.is_pro` + `pro_until` → `setPro()`; `openProModal()` Pro tanıtım/ödeme akışı
  (Lemon Squeezy entegrasyonu backlog'da).
- **Oturum kalıcılığı:** `initAuth()` açılışta `navigator.storage.persist()` (iOS/Safari'nin oturumu erken
  silmesini azaltır) + `visibilitychange` ile öne gelince `supa.auth.getSession()` (uzun arka plandan sonra
  tazeler). Supabase Auth → Sessions: **Inactivity timeout = 168 saat (7 gün)**, Time-box = never,
  Single-session = off. (Erken "düz giriş" sorununun kök sebebi kısa inactivity timeout'tı → 7 güne çıkarıldı.)
- **onSession optimizasyonu:** Giriş sonrası **tek profil sorgusu** (`is_pro,pro_until,status,vehicle,is_admin,lang`
  bir kerede) + `favorites`/`visits` **paralel** (`Promise.all`) → eski 6 sıralı sorgu yerine 3 paralel.
  `refreshEntitlement/loadVehicleRemote/loadAdminFlag/loadLangRemote` opsiyonel `prof` parametresi alır
  (verilince sorgu atlanır; parametresiz çağrılar kendi sorgusunu yapar — bozulmaz).

  uydu görünümü neredeyse aynı, zoom çok daha hızlı. **sw `TILES` v1→v2 bump'ı** activate'te eski 1024px uydu
  tile'larını temizler (`k!==CACHE && k!==TILES` filtresi).
